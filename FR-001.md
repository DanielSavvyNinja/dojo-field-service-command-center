# FR-001 — dojo-field-service-command-center (Dispatch Board widget)

This is the dispatch board UI that Daniel was reviewing. The free-text fields he flagged live here.

## Scope

### 1. Customer / Contact picker (FR-001 #1)
- Replace any free-text customer name/email/phone/address inputs on the job creation/edit form with a Contact picker.
- Picker: typeahead against the customers table (which mirrors GHL contacts via the connector).
- "New customer" inline action that creates a row in customers (the connector pushes it to GHL on next sync).
- Display fields on the dispatch card pull from the FK relationship, NOT from the denormalized customer_name etc. on jobs (those stay as a read cache for backwards compat — don't remove them yet, just stop reading them in this UI).

### 2. Service Type picker (FR-001 #2)
- Replace free-text service type inputs with a dropdown sourced from price_book table.
- Picker shows: name, base price, duration, taxable/non-taxable badge.
- Selecting an item populates job_line_items.price_book_id AND keeps service_id (text) in sync for legacy code.
- Multi-line jobs supported (multiple job_line_items).

### 3. Technician picker (FR-001 #3)
- Tech assignment dropdown pulls from technicians table where is_active=true.
- Roster updates automatically as the connector syncs from GHL Team API (15-min cadence).
- No "manual entry" — if a tech isn't in GHL, they shouldn't be assignable.

### 4. Create Invoice button (FR-001 #4)
- New button on the job detail panel: "Create Invoice".
- Action: POST to connector /api/invoices with {job_id}. Connector creates the local invoice row, pushes to GHL, returns invoice ID + GHL invoice URL.
- After creation: button changes to "View Invoice" linking to GHL's hosted invoice page.

### 5. Collect Payment button (FR-001 #4)
- New button on the job detail panel: "Collect Payment".
- Three modes:
  - "Send payment link" → connector returns hosted GHL payment URL → copy to clipboard or email/SMS to customer.
  - "Card on file" → if customer has a saved method in GHL, charge directly.
  - "Manual cash/check" → dialog: amount, method, notes → connector creates payment row with status='paid'.
- Payment status badge on dispatch cards: Unpaid (red) / Partial (yellow) / Paid (green).

## Files most likely to touch
